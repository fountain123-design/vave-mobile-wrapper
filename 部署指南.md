# VAVE AI 移动应用 - Codemagic 部署指南

## 文件说明

| 文件 | 作用 |
|------|------|
| `lib/main.dart` | Flutter 主代码，WebView 加载 VAVE 应用 |
| `pubspec.yaml` | Flutter 依赖配置 |
| `codemagic.yaml` | CI/CD 工作流配置 |
| `android/` | Android 平台配置 |
| `ios/` | iOS 平台配置 |

## 部署步骤

### 第一步：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 仓库名：`vave-mobile`（或其他）
3. 选择 **Public** 或 **Private**
4. 不要勾选 "Initialize this repository with a README"
5. 点击 **Create repository**

### 第二步：上传代码

```bash
# 解压 vave-mobile-wrapper.zip 后进入目录
cd vave-mobile-wrapper

# 初始化 git
git init
git add .
git commit -m "Initial commit: VAVE mobile wrapper"

# 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/vave-mobile.git

# 推送代码
git push -u origin main
```

### 第三步：Codemagic 连接仓库

1. 登录 https://codemagic.io
2. 点击 **"Add application"**
3. 选择 **GitHub** → 授权访问 → 选择 `vave-mobile` 仓库
4. 点击 **"Add application"**

### 第四步：配置 Android 签名（必需）

1. 生成 keystore 文件：
```bash
keytool -genkey -v -keystore vave.keystore -alias vave -keyalg RSA -keysize 2048 -validity 10000
```
- 记住你设置的密码！

2. 在 Codemagic 控制台：
   - 进入你的应用 → **Settings** → **Code signing**
   - 点击 **Android keystores** → **Add keystore**
   - 上传 `vave.keystore` 文件
   - 填写 keystore password、key alias、key password
   - Keystore reference name 填：`vave_keystore`

3. 修改 `codemagic.yaml` 中的邮箱：
   - 把 `your-email@example.com` 换成你的真实邮箱

### 第五步：触发构建

1. 修改 `lib/main.dart` 中的 `VAVE_APP_URL`：
```dart
// 临时地址（会变化）
const String VAVE_APP_URL = 'https://fruity-rules-mix.loca.lt';

// 改成你的固定域名（部署后）
const String VAVE_APP_URL = 'https://your-vave-app.vercel.app';
```

2. 提交并推送：
```bash
git add lib/main.dart codemagic.yaml
git commit -m "Update app URL and email"
git push
```

3. Codemagic 自动开始构建！

### 第六步：下载构建产物

构建完成后：
- 进入 Codemagic → 你的应用 → **Builds**
- 点击最新构建 → **Artifacts**
- 下载：
  - `app-release.apk`（Android 安装包）
  - `Runner.app`（iOS，未签名）

## iOS 真机安装（需要 Apple Developer）

如果没有 Apple Developer 账号（¥688/年）：
- iOS 只能构建出未签名的 `.app`
- 无法安装到真机或上架 App Store

有账号的话，在 Codemagic 上传：
- iOS 证书（.p12）
- Provisioning profile

然后重新构建，就能拿到签名的 IPA 文件。

## 常见问题

**Q: 构建失败，提示 "Could not find keystore"？**
A: 检查 Codemagic 控制台是否正确上传了 keystore，reference name 是否匹配 `codemagic.yaml` 中的 `vave_keystore`。

**Q: iOS 构建成功但无法安装？**
A: 未签名的 iOS 应用只能运行在模拟器。真机需要 Apple Developer 账号 + 正确配置签名。

**Q: 应用打开后显示空白？**
A: 检查 `VAVE_APP_URL` 是否可访问，是否允许 HTTP（需要 HTTPS 或配置 ATS）。

**Q: 如何更新应用？**
A: 修改代码 → `git commit` → `git push`，Codemagic 自动重新构建。
